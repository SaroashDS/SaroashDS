## What I Build

I work where the model meets the business process: requirements, data, interfaces,
automation, deployment, and adoption. My favorite problems are the ones that need
someone to sit with the workflow, understand the edge cases, architect the system,
and stay close enough to ship.

```text
Business ambiguity -> AI architecture -> prototype -> integration -> deployment -> iteration
```

- AI systems that automate real operational workflows.
- Forward-deployed prototypes that move from demo to usable product.
- Computer vision pipelines for real-world detection and prediction.
- Data products that make decisions easier for non-technical users.
- Web apps that make AI capabilities accessible, inspectable, and fast.

## Operating Mode

I think like an architect and move like an engineer.

- Map the real workflow before choosing the model.
- Build the smallest useful system, then harden the path that users actually touch.
- Keep the interface close to the intelligence: AI is only useful when people can use it.
- Design for integration from day one: APIs, data contracts, auth, observability, and deployment.
- Stay forward deployed: learn from users, ship changes, and close the gap between prototype and production.
